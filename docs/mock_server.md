# モックサーバー (json-server)

## 概要

このプロジェクトでは、バックエンドAPIが未完成の場合や、通信制限のある環境でも開発を止めないために、**`json-server`** を使ったローカルモックサーバー環境を導入しています。

アプリのソースコード（Dart）を一切書き換えることなく、接続先の設定（Flavor）を切り替えるだけで、手元のパソコン内で動作する擬似的なAPIと通信できます。

---

## 🛠 技術スタック

- **ツール**: [json-server](https://github.com/typicode/json-server) (安定版 0.17.4 を推奨)
- **データ形式**: JSON

---

## 📁 ディレクトリ構成

モックサーバーに関連するファイルは、プロジェクトルートの `mock/` ディレクトリに集約されています。

```plaintext
mock/
 ├── db.json          # データの「倉庫」。APIが返す実データを定義
 ├── routes.json      # パスの「案内図」。カスタムURL（/auth/login等）の紐付け
 └── start.sh         # 起動用スクリプト
```

---

## 🚀 使い方

### 1. サーバーの起動

ターミナルで以下のコマンドを実行します。

```bash
./mock/start.sh
```

成功すると、`http://localhost:3000/users` などのエンドポイントが有効になります。

### 2. アプリの起動

Flutterアプリを **`local` フレーバー** で起動してください。
`config/flavor_local.json` の `BASE_URL` が `http://localhost:3000` に設定されているため、自動的にモックサーバーへ接続されます。

---

## ✨ このモックサーバーでできること

単に固定値を返すだけでなく、**実際のAPIに近い「振る舞い」** をシミュレートできます。

### 1. データの永続化 (CRUD対応)

- **GET**: データの取得
- **POST**: 新しいデータの追加
- **PATCH / PUT**: 既存データの更新
- **DELETE**: データの削除

例えば、アプリから「ユーザー追加（POST）」を実行すると、`mock/db.json` の中身が自動的に書き換えられ、次回の取得（GET）時には追加されたデータが含まれるようになります。

### 2. 特殊なパスの紐付け (Custom Routes)

`routes.json` を使用することで、以下のようなネストされたパスやカスタムパスにも対応しています。

- `/auth/login` → `db.json` の `login` データを返す
- `/v1/users` → `db.json` の `users` リストを返す（将来的な拡張例）

---

## 💡 開発のヒント

- **データの初期化**: `db.json` を直接編集することで、いつでもテスト用データを準備・リセットできます。
- **Androidエミュレータでの注意**: Androidエミュレータから接続する場合、自身のパソコン（localhost）を指すIPアドレスは `10.0.2.2` になります。必要に応じて `config/flavor_local.json` を調整してください。
- **意図的な遅延**: 起動コマンドに `--delay 2000` を追加することで、わざと2秒の遅延を発生させ、ローディング画面のテストなどが可能です。
