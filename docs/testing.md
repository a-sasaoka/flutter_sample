# 🧪 テスト方針とゴールデンテスト (Testing & Golden Tests)

このプロジェクトでは、アプリが正しく動くこと、そして画面の見た目が崩れていないことを確認するために、自動テストを導入しています。

---

## 📌 テストの種類

アプリの品質を守るために、以下のテストを実行しています。

1. **単体テスト (Unit Test)**
   - プログラムの計算処理やデータの変換など、細かい機能が正しく動くかを個別に確認するテストです。
2. **ウィジェットテスト (Widget Test)**
   - ボタンがタップできるか、文字が正しく表示されるかなど、画面の部品の動きを確認するテストです。
3. **ゴールデンテスト (Golden Test) 🌟**
   - 画面の「お手本の写真（スクリーンショット）」をあらかじめ保存しておき、新しくプログラムを書き換えたときに、画面が崩れてしまっていないかを写真同士を重ね合わせて自動で比べるテストです。

---

## 🎨 ゴールデンテスト (Alchemist)

見た目の崩れを防ぐために、**`alchemist`** という最新のパッケージを使用しています。
テスト用の日本語フォント（Noto Sans JP）を読み込んで比較するため、日本語の文字化けを起こさずに綺麗な写真でテストができます。

### 対象の画面

プロジェクト内のすべての主要な画面や共通ダイアログ（合計11箇所）にゴールデンテストを導入しています。

- **メイン機能**: ホーム画面、メモ画面、AIチャット画面、設定画面
- **ユーザー一覧**: ユーザー一覧画面
- **認証**: ログイン画面、新規登録画面、メール認証画面、パスワード再設定画面
- **その他**: 404エラー画面、バージョンアップ通知ダイアログ

---

## 🛠️ テストの実行コマンド

テストを実行する際は、バージョンを固定するために、必ず `fvm` をコマンドの先頭につけて実行します。

### 通常のテスト実行 (見た目や機能のチェック)

```bash
fvm flutter test
```

### ゴールデン画像（お手本写真）の作成・更新

画面のデザインを新しく変更したときや、新しい画面を追加したときは、以下のコマンドを実行してお手本の写真を新しく撮影し直します。

```bash
fvm flutter test --update-goldens
```

※新しく撮影された写真は、それぞれのテストファイルがあるフォルダの `goldens/` 内に保存されます。

---

## 🤖 CI (GitHub Actions) との連携

プログラムを GitHub にアップロード（プッシュ）するたびに、サーバー上で自動的にテストが実行されます。
`alchemist` は開発環境（macOS）とサーバー環境（Linux）で文字の太さなどが微妙に変わってテストが失敗してしまうのを防ぐため、それぞれのOSに対応したお手本画像で検証する設定になっています。
