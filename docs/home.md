# ホーム画面 (HomeScreen)

## 概要

ホーム画面は、アプリの主要機能へのアクセスを提供するセントラルハブとしての役割を持ちます。\
また、開発者向けツールやアプリ情報の確認、クラッシュテストなどのデバッグ機能も集約されています。

---

## 主な機能と構成

最新の UI デザイン基準に基づき、情報は 3 つの主要なカテゴリ（Card）に整理されています。

### 1. メイン機能 (Main Features)

AI チャット、グラフ、メモ帳、ユーザー一覧などの主要な機能モジュールへのナビゲーションを提供します。

- 各項目は `ListTile` と一貫性のあるアイコンで構成されています。
- 認証状態（Firebase利用有無）に応じて、パスワードリセット画面などの表示が動的に切り替わります。

### 2. 開発・デバッグツール (Developer Tools)

アプリの品質管理やデバッグを補助する機能です。

- **統合ログ (TalkerScreen)**: 通信履歴やエラーログをアプリ上で確認できます（`prod` 環境以外で表示）。
- **クラッシュテスト**: Firebase Crashlytics が正しく動作するかを検証するための強制クラッシュ機能。
- **分析テスト**: Firebase Analytics へのイベント送信をテストします。

### 3. アプリ情報 (App Information)

実行環境（Flavor）やパッケージ情報の確認が可能です。

- **環境チップ**: 現在の Flavor（LOCAL, DEV, STG, PROD）を視覚的に判別できます。
- **PackageInfo 連携**: ボタンタップにより、アプリ名や Bundle ID を OS から取得して表示します。

---

## 状態管理とロジック

### バージョンアップ通知の統合

`updateRequestControllerProvider` を監視し、新しいバージョンが必要な場合に `VersionUpDialog` を自動的に表示します。

- `ref.listen` を使用して、アプリ起動中やフォアグラウンド復帰時の状態変化をリアルタイムに検知します。
- 一度「後で」を選択した場合は、セッション中（あるいは永続化状態に応じて）再表示を抑制する制御が行われています。

### 堅牢な UI 描画

アップデート情報の取得に失敗（`AsyncError`）した場合でも、メインのメニュー表示を阻害しないよう、エラーハンドリングとフォールバック描画が徹底されています。

## テスト

この機能には、以下の包括的なテストが含まれています。

- **Widget Test**:
  - 各メニュー項目の表示とナビゲーションの動作。
  - Flavor ごとのメニューの出し分け。
  - アップデートダイアログの表示パターン（オプション／必須／キャンセル済み）。
  - エラー発生時の UI 継続性。
  - スクロールを伴うタップ操作の安定性（ビューポート固定による検証）。
- **Golden Test**:
  - `HomeScreen`: ライトモード/ダークモードでの画面全体の表示確認。
